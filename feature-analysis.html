01TL;DR

Короткая сводка: что работает, где критические пробелы и где должен быть фокус.

Ядро работает — создание турнира, публичная регистрация по slug, single-elimination, drag-drop, ввод счёта по сетам, роли owner/editor, 3 языка, dark theme, realtime.

Но заявлено и не реализовано: live-scoring по очкам (нет useTennisScoring.js и live_scores), роль counter, профили игроков с историей, связь клуб ↔ турнир (tournaments.club_id не используется).

Архитектурная дыра: нет сущности players — всё в entry_members строками. Без этого невозможны профили, ELO, история.

Топ-5 рекомендаций на 3–6 месяцев

  1. Закрыть P0-долг (секция 3) — иначе всё дальнейшее строится на песке.
  2. Публичная страница клуба /clubs/:slug и связать турниры с клубами.
  3. Round-robin + group stage + playoff — group stage отсутствует в FEATURE_RESEARCH, но самый востребованный в клубах.
  4. Категории участников (возраст / пол / уровень) + waiting list — ожидание любого серьёзного клуба.
  5. Telegram-бот — самая дешёвая killer-фича для RU / LT / EE рынка.
